## 1. 战略愿景与第一性原理 (Strategic Vision & First Principles)

### 1.1 终极愿景
**"The Bloomberg & Moody's for Data Assets"**
DataPricing Ai 不仅仅是一个估值计算器，它是全球数据资产的**定价中心**、**信用评级机构**和**流通基础设施**。我们通过连接数据持有者、学术界/金融界专家和资本市场，解决数据要素流通中最大的痛点——**定价共识**。

### 1.2 第一性原理深度解析
我们回归事物的本质，重新定义产品逻辑：

1.  **信任源于透明 (Trust through Transparency)：**
    *   *问题：* 传统的“AI 估值”是黑盒，银行和审计无法采信。
    *   *解法：* **“白盒计算”原则**。AI (LLM) 仅作为语义理解层，负责提取元数据；核心估值逻辑由开源、可审计的 TypeScript 数学引擎执行。

2.  **权威源于分权 (Authority through Decentralization)：**
    *   *问题：* 没有一种算法能通过所有行业的验收。
    *   *解法：* **“专家模型市场” (Model Marketplace)**。平台不垄断定价权，而是引入清华大学、斯坦福、四大行、数据交易所等权威机构发布模型。用户选择谁的模型，就是选择谁的信用背书。

3.  **价值源于流动 (Value through Flow)：**
    *   *问题：* 静态的估值报告是一张废纸。
    *   *解法：* **“全生命周期监测” (Lifecycle Monitor)**。数据价值随市场供需实时波动，我们提供类似股票 K 线图的价值趋势监控，并直接打通交易所 API。

## 3. 核心功能模块详解

### 3.1 智能访谈官 (AI Semantic Layer)
*   **技术栈：** Google Gemini 1.5 Pro/Flash + Prompt Engineering。
*   **功能：**
    *   自然语言交互，降低专业门槛。
    *   **意图识别引擎：** 自动判断用户是为了“融资”（侧重收益法）、“交易”（侧重市场法）还是“入表”（侧重成本法）。
    *   **线索提取 (ETL)：** 从对话中结构化提取六维指标（质量、稀缺性、应用、合规、热度、成本）。

### 3.2 专家模型市场 (The Marketplace)
*   **核心逻辑：** `Asset_Value = Base_Algorithm * Expert_Model_Coefficient`
*   **市场机制：**
    *   **Stanford Digital Lab 模型：** 激进型，适合一级市场融资（高乘数）。
    *   **ICBC 银行风控模型：** 保守型，适合银行抵押贷（重合规，低风险偏好）。
    *   **上海数交所标准模型：** 均衡型，适合挂牌交易。
*   **商业闭环：** 用户支付 DV Points 购买模型使用权 -> 平台抽佣 -> 专家获利。

### 3.3 白盒定价引擎 V2 (White-box Engine)
*   **设计原则：** 代码即法律 (Code is Law)。
*   **计算公式示例（TypeScript 实现）：**
    ```typescript
    // 市场法 + 专家系数修正
    FinalPrice = (Industry_Base_Rate * Market_Multiplier) 
                 * (1 + Scarcity_Score / 200) 
                 * Expert_Model_Risk_Factor;
    ```
*   **输出：** 估值区间、计算方法论说明、关键驱动因子分析。

### 3.4 资产驾驶舱 (Asset Dashboard)
*   **功能：**
    *   **雷达图谱：** 实时展示资产的六维健康度。
    *   **价值 K 线：** 模拟展示资产过去 12 个月的价值波动。
    *   **市场行情 (Ticker)：** 滚动展示行业数据成交动态（如：“医疗数据本周上涨 12%”）。

## 4. 技术架构与安全性

### 4.1 前端架构
*   **框架：** React 19 + TypeScript (保证类型安全)。
*   **UI 系统：** Tailwind CSS (极简、高性能)。
*   **可视化：** Recharts (用于雷达图和趋势图)。
*   **状态管理：** React Hooks + Context API。

### 4.2 数据安全与隐私 (Code-to-Data)
*   **隐私红线：** 用户的**原始数据**（CSV/Excel）绝不上传至 LLM。
*   **处理流程：** 
    1.  浏览器端计算数据的元数据（Metadata，如行数、列名、空值率）。
    2.  仅将**元数据**发送给 AI 进行语义分析。
    3.  估值计算在本地或受控沙箱中完成。
